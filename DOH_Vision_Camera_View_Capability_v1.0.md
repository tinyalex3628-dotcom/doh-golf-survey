# DOH Vision Camera View Capability & Limits v1.0
**Project C — 카메라 앵글별 "측정 가능 / 근사 / 불가" 명시 (정직성 계약)**
*작성일: 2026-07-01 / 상태: SPEC DRAFT (프로 검토 전)*
*상위: Coordinate System Spec v1.0 §7 · Technical Architecture v1.1 (Confidence/Error)*

> **원칙: 안 되는 건 대충 하지 않는다.**
> 카메라 한 대는 렌즈를 향한 **깊이축(depth)을 측정하지 못한다.** 이건 알고리즘으로
> 극복되지 않는 물리적 한계다. 이 문서는 각 앵글에서 **무엇이 되고 무엇이 안 되는지**를
> 못박아, "안 되는 값을 그럴듯한 숫자로 지어내는" 일을 금지한다.
>
> **강제 규칙:**
> - ✕(불가) 지표는 **값을 만들지 않는다 → `null`.** 0이나 추정치로 채우지 않는다.
> - ≈(근사) 지표는 **값을 내되 "참고용"으로 표시**하고 절대 정밀값으로 취급하지 않는다.
> - ✅(가능) 지표만 진단 근거로 정상 사용한다.

---

## 1. 왜 앵글마다 다른가 (물리)

카메라는 2D 센서다. 3D의 한 축(카메라를 향하는 깊이)이 눌려서 사라진다.
포즈 모델의 world 3D도 이 깊이를 **추정**할 뿐, 실제로 잰 게 아니다.

```
FO(정면): 렌즈축 = 골퍼의 앞뒤(전후)   → 앞뒤(전후) 깊이 소실
DTL(측면): 렌즈축 = 목표선(비구선) 방향 → 목표선 깊이 소실
```
→ **소실된 축 방향의 움직임은 측정 불가/부정확.** 그 축에 수직인 움직임만 신뢰.

---

## 2. FO (Face-On, 정면) 카메라

**골퍼의 정면(가슴)을 바라봄. 목표선이 화면에서 좌우로 흐름.**

| 측정 | 등급 | 이유 |
|---|:--:|---|
| 체중이동 / 스웨이(좌우) | ✅ 가능 | 목표선 방향 = 화면 좌우, 잘 보임 |
| 힙 슬라이드(좌우) | ✅ 가능 | 좌우 병진 |
| 머리 좌우/상하 움직임 | ✅ 가능 | 프레임 내 위치 |
| 척추 **좌우** 기울기 | ✅ 가능 | 정면 기울기 |
| 리버스 피벗 / 행잉백 | ✅ 가능 | 좌우 체중편향 |
| 무릎각 | ✅ 가능 | 관절 사이각 |
| 어깨/골반 회전량 | ≈ 근사 | 회전 시 어깨선이 앞뒤로 도는데 그 깊이가 추정 → 절대각 부정확 |
| X-Factor(분리각) | ≈ 근사 | 상동 |
| 척추 **전후** 굴곡(숙임) | ✕ 불가 | 앞뒤 축 = 렌즈 방향, 소실 |
| 얼리 익스텐션(골반 전후) | ✕ 불가 | 앞뒤 병진 소실 |
| 핸드 뎁스(앞뒤) | ✕ 불가 | 앞뒤 소실 |
| **샤프트각 / 스윙플레인** | ✕ 불가 | 샤프트가 렌즈 향해 투영, 각도 왜곡 |
| **클럽패스(인-아웃)** | ✕ 불가 | 인-아웃 = 앞뒤 축 |
| **클럽페이스각** | ✕ 불가 | 앞뒤 성분 |

**FO 한 줄 요약:** *좌우·상하·체중은 잘 본다. 앞뒤(플레인·샤프트·패스·전후굴곡)는 못 본다.*

---

## 3. DTL (Down-The-Line, 측면·비구선 뒤) 카메라

**골퍼 뒤에서 목표선을 따라 바라봄. 목표선이 화면 안쪽으로 들어감.**

| 측정 | 등급 | 이유 |
|---|:--:|---|
| **샤프트각 / 스윙플레인** | ✅ 가능 | DTL 전용. 샤프트가 옆으로 보임 |
| **클럽패스(인-아웃 경향)** | ✅ 가능 | 플레인 상 궤적 |
| 척추 **전후** 굴곡(자세각) | ✅ 가능 | 옆에서 숙임 각 보임 |
| 얼리 익스텐션(골반→볼) | ✅ 가능 | 앞뒤 병진이 화면 좌우로 보임 |
| 테이크어웨이 궤적 / 온플레인 | ✅ 가능 | 플레인 이탈 |
| 오버더톱 | ✅ 가능 | 다운스윙 손 경로 |
| 무릎각 | ✅ 가능 | 관절 사이각 |
| 어깨/골반 회전량 | ≈ 근사 | 회전 깊이 추정 → 절대각 부정확 |
| X-Factor | ≈ 근사 | 상동 |
| 머리 **좌우** 스웨이 | ≈ 근사 | 좌우 = 목표선 깊이 성분 일부 소실 |
| 체중 좌우 배분 | ≈ 근사 | 좌우 축 일부 소실 |
| 척추 **좌우** 기울기 | ✕ 불가 | 좌우 기울기 = 목표선 방향 소실 |

**DTL 한 줄 요약:** *플레인·샤프트·패스·전후자세는 잘 본다. 순수 좌우(좌우스웨이·좌우틸트)는 약하거나 못 본다.*

---

## 4. 종합 매트릭스 (한눈에)

| 측정 대상 | FO | DTL | 2-cam(FO+DTL) |
|---|:--:|:--:|:--:|
| 스웨이/체중(좌우) | ✅ | ≈ | ✅ |
| 척추 좌우틸트 | ✅ | ✕ | ✅ |
| 척추 전후굴곡 | ✕ | ✅ | ✅ |
| 얼리 익스텐션 | ✕ | ✅ | ✅ |
| 어깨/골반 회전, X-Factor | ≈ | ≈ | ✅(정밀↑) |
| 무릎각 | ✅ | ✅ | ✅ |
| 샤프트각/플레인 | ✕ | ✅ | ✅ |
| 클럽패스/페이스 | ✕ | ✅ | ✅ |

**결론:**
- **회전·X-Factor는 어느 단일 카메라로도 "근사"가 최선.** 정밀 절대각은 2-cam 또는 3D캡처/센서 필요. → 정직하게 "참고용"으로 표기.
- **단일 카메라라면 DTL이 커버리지 넓음**(플레인·샤프트·전후자세 + 무릎). 좌우(스웨이·좌우틸트)만 포기.
- **최선은 FO+DTL 2대.** 서로의 소실축을 보완 → 거의 전 항목 ✅.

## 5. 코드 반영 계약 (analyzer가 지켜야 할 것)

| 등급 | UI 표기 | 값 처리 | DOH 전달 |
|---|---|---|---|
| ✅ 가능 | ✅정확 | 정상 산출 | 정상 근거 |
| ≈ 근사 | ≈근사(참고용) | 값 내되 참고 표시 | confidence 낮춤 + flag |
| ✕ 불가 | ✕측정불가 | **null (지어내지 않음)** | 전달 안 함(null) |

- `analyzer2.html` 는 선택된 뷰(FO/DTL)에 따라 위 등급을 배지로 표시하고, ✕는 값을 만들지 않는다.
- 이 표는 Coordinate System Spec §7의 `c_view` 및 Feature Spec의 `view` 컬럼과 일치해야 한다.

## 6. 촬영 가이드 (회원 안내용)

- **1대만 가능하면 DTL(측면, 비구선 뒤)** 로 찍으세요. 삼각대 수평, 허리 높이, 골퍼 전체가 프레임에.
- **정밀 진단을 원하면 FO+DTL 2번** 찍으세요(같은 스윙 아니어도 됨, 각 뷰의 강점 항목만 사용).
- 고정 삼각대 권장(손각도 흔들리면 좌표 노이즈↑). 가능하면 **120fps↑**(임팩트 구간).
- 배경은 단순하게(샤프트 Hough 오검출↓).
